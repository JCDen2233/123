# SKILL: Isometric Multiplayer Game (Python + Web)

## Описание
Многопользовательская изометрическая 2D-игра с бэкендом на Python (Flask + Flask-SocketIO, порт 5000) и фронтендом на HTML5 Canvas + JavaScript. Поддержка реального времени через WebSocket.

## Технический стек

| Компонент | Технология |
|-----------|------------|
| Backend | Python 3.x + Flask + Flask-SocketIO |
| Frontend | HTML5 Canvas + Vanilla JS + CSS3 |
| Real-time | Socket.IO (WebSocket) |
| Рендеринг | Изометрическая проекция (математические преобразования) |
| Анимация | JavaScript requestAnimationFrame |
| Хранение | JSON файлы / SQLite (опционально) |

## Математика изометрии

### Прямое преобразование (grid → screen)
```
screen_x = (grid_x - grid_y) * TILE_W / 2 + OFFSET_X
screen_y = (grid_x + grid_y) * TILE_H / 2 + OFFSET_Y
```

### Обратное преобразование (screen → grid)
```
grid_x = ((screen_x - OFFSET_X) / TILE_W + (screen_y - OFFSET_Y) / TILE_H)
grid_y = ((screen_y - OFFSET_Y) / TILE_H - (screen_x - OFFSET_X) / TILE_W)
```

### Стандартные размеры тайлов
- TILE_W = 64 (ширина тайла)
- TILE_H = 32 (высота тайла)
- Соотношение 2:1 для классической изометрии

## Сетевая архитектура

### Серверная авторитетная модель
Сервер — единственный источник истины. Клиенты отправляют действия, сервер валидирует и рассылает обновлённое состояние.

### Socket.IO события

#### Клиент → Сервер
| Событие | Данные | Описание |
|---------|--------|----------|
| `join` | `{nickname, color}` | Подключение игрока |
| `move` | `{dx, dy, timestamp}` | Направление движения |
| `action` | `{type, target}` | Игровое действие |
| `chat` | `{message}` | Сообщение в чат |

#### Сервер → Клиент
| Событие | Данные | Описание |
|---------|--------|----------|
| `init` | `{players, map, you_id}` | Начальное состояние при подключении |
| `state` | `{players: [{id, x, y, ...}]}` | Обновление позиций всех игроков (10 tick/sec) |
| `player_joined` | `{id, nickname, x, y}` | Новый игрок |
| `player_left` | `{id}` | Игрок вышел |
| `chat` | `{id, nickname, message}` | Сообщение чата |
| `action_result` | `{type, data}` | Результат действия |

### Интерполяция на клиенте
Клиент хранит предыдущее и текущее состояние позиций других игроков и интерполирует между ними для плавного движения:

```
renderX = prevX + (currentX - prevX) * interpolationFactor
renderY = prevY + (currentY - prevY) * interpolationFactor
```

### Client-side prediction (свой игрок)
Локальный игрок двигается мгновенно на клиенте, сервер корректирует позицию при расхождениях.

## Игровой цикл сервера

```
10 ticks/second:
  1. Обработать входящие действия игроков
  2. Обновить позиции (валидация + коллизии)
  3. Обновить NPC/врагов (ИИ)
  4. Обработать события мира
  5. Рассылка state всем подключённым клиентам
```

## Правила кода

### Backend (Python)
- Flask-SocketIO для WebSocket коммуникации
- Сервер — авторитетный источник состояния
- Валидация ВСЕХ входящих данных от клиентов
- Rate limiting действий (защита от спама)
- JSON для всех данных
- Thread-safe операции с состоянием (threading.Lock)

### Frontend (JavaScript)
- Игровой цикл через requestAnimationFrame
- Socket.IO client для сетевых событий
- Интерполяция позиций других игроков
- Client-side prediction для своего игрока
- Delta-time для плавной анимации
- Все константы в начале файлов
- Разделение логики и рендеринга

### CSS
- Минималистичный стиль без фреймворков
- Canvas на весь экран
- HUD поверх canvas через absolute positioning
- Чат с прокруткой поверх canvas

## Команды запуска

```
pip install flask flask-socketio
```

## Приоритет разработки

1. Сервер + Canvas → базовая инфраструктура
2. Изометрическая сетка → визуальная основа
3. Socket.IO подключение → сетевая база
4. Перемещение игрока + синхронизация → мультиплеер
5. Коллизии → игровая логика
6. Камера → навигация по большим картам
7. Спрайты → визуальное оформление
8. HUD + чат → интерфейс
9. Интерполяция → плавность мультиплеера
10. Интерактивность → глубина геймплея
